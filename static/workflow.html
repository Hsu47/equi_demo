Equi PDF Extraction Pipeline

New PDF → Extract → Validate → Score → Signal  ·  end-to-end < 100ms

1
Raw Extraction — read the PDF ~30ms

pdfplumber.open()

Open PDF, iterate pages. Extract two things from each page:

→ tables[]   structured 2D arrays (if PDF has real tables)
→ full_text   raw text layer concatenated across pages
→ page_char_counts[]   chars per page (for OCR detection)
2
Metadata Extraction — what fund is this? ~5ms

Fund Name

Scan for "Fund LP", "Capital Fund" etc. in short header lines

→ "Meridian Global Macro Fund LP"

AUM / Ending NAV

6-layer parser: European decimal → abbreviated ($2.66M) → full number → ISO code suffix → bare number

→ 2.66 (millions)

Beginning NAV

Same 6-layer parser, different label keywords ("beginning nav", "opening balance")

→ 2.45 (millions)

Currency

3 detection layers:
1. Explicit: "Fund Currency: EUR"
2. AUM context: € near "Net Assets"
3. Symbol frequency: count $/€/£/¥

→ {USD, explicit}

Fee Structure

Pattern matching:
"Management Fee: 1.5%" → explicit
"2 and 20" → shorthand

→ {mgmt: 1.0%, perf: 10%}

OCR Detection

Check avg chars/page. Below 80 → likely scanned image, not text PDF. Flag for confidence penalty.

→ ocr_needed: false
3
Monthly Returns Extraction — cascading fallback ~15ms

Try each method top-to-bottom. Stop at the first one that finds ≥ 3 months. More reliable methods are tried first — if the best method works, we never fall through to weaker ones.

①
table
+0.400
Find header row → locate "Net Return" column by index → extract value from each month row
pdfplumber finds table structure
▼ < 3 months? fall through
②
calendar_text
+0.340
Find "Jan Feb Mar..." header → parse year rows → trailing 12 months across years
raw text has month grid layout
▼ < 3 months? fall through
③
text
+0.200
Line-by-line: if line has month name + percentage number → grab it
unstructured text with month-return pairs
▼ < 3 months? fall through
④
summary
+0.120
Find "1 Month: 2.4%, YTD: 8.1%" — only summary periods, no monthly series
factsheet with period returns only
▼ nothing found → raise ValueError("Insufficient return data")
4
Validation — can we trust these numbers? ~2ms

NAV Reconciliation

implied = begin × (1+r₁)(1+r₂)...(1+r₁₂)

Compare implied ending NAV vs stated ending NAV.
δ < 5% → ✓ reconciled
δ ≥ 5% → ✗ wrong column, missing months, or sign error
No begin NAV → skipped (neutral)

Return Type Classification

Two signals, combined:

Signal 1: Column header
"Net Return" → net   "Gross Return" → gross

Signal 2: Document text
"net of fees" → net   "before fees" → gross

Neither → unknown (confidence −0.15)

5
Confidence Score — 6-dimensional weighted sum <1ms

confidence = clamp(0, 1, method×0.4 + completeness×0.5 + warnings×0.1 + recon_adj + rt_adj + ocr_adj)

Method
table=1.0 · cal_text=0.85 · text=0.5 · summary=0.3
× 0.4
Completeness
months_found / 12 (capped at 1.0)
× 0.5
Warnings
each warning costs −0.10 from this component
× 0.1

pass → +0.05 · fail → −0.20 · skipped → 0
± adj
Return Type
net → 0 · gross → −0.10 · unknown → −0.15
− adj

− adj

Design Rationale

Why cascading fallback? — Real LP reports come in hundreds of formats. No single parser handles all of them. By trying the most reliable method first and falling through only on failure, we maximize accuracy while maintaining coverage.

Why 6-dimensional confidence? — A single "pass/fail" hides too much. A PDF could have perfect table extraction (method=1.0) but unknown return type (−0.15) and no reconciliation (0.00). The breakdown lets LPs audit exactly where the uncertainty lives.

Why reconciliation is ±, not just +? — Passing reconciliation is mild confirmation (+0.05). Failing is a strong red flag (−0.20) because it means the extracted returns don't compound to the stated NAV — something is wrong. The asymmetry reflects that a failed cross-check is much more informative than a passed one.

Why raw_returns[] is the universal interface? — Transform and Score never know if the data came from Bloomberg, CSV, or PDF. This means adding a new data source (email parser, API connector) only requires implementing one function that outputs raw_returns[]. Everything downstream — Sharpe, Sortino, Correlation, Composite Score — works automatically.
